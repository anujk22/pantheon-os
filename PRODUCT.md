# Product

## Register

product

## Users

Pantheon OS is for a local-first power user who wants one place to collect work, research, chat history, files, tasks, project context, and personal operating memory. The user is often switching between AI providers, school/work systems, local tools, calendars, repositories, and long-running projects.

## Product Purpose

Pantheon OS exists to turn raw personal context into useful local structure. Success means the user can capture anything, inspect it, triage it into tasks/cases/artifacts/memory, and later retrieve or act on it without relying on a cloud service or trusting unreviewed automatic memory.

## Brand Personality

Capable, calm, exacting. The mythological Emerald Pantheon aesthetic should feel premium, but the product must behave like a serious operating surface: dense where useful, honest about system state, and free of fake agent activity.

## Anti-references

Avoid SaaS dashboards that imply capability with decorative charts, fake statuses, staged agent activity, or hardcoded productivity metrics. Avoid chatbot wrappers where memory is an unreviewed side effect. Avoid interfaces that look impressive in screenshots but do not persist, classify, or execute real work.

## Design Principles

1. Capture before committing: raw context enters an inbox before it becomes memory, tasks, artifacts, or cases.
2. Local truth beats simulated intelligence: show real persisted data, empty states, and system state instead of mock agent behavior.
3. Triage is the core workflow: every intake surface should make review and destination selection fast.
4. Autonomy needs auditability: background action should leave records the user can inspect and correct.
5. Aesthetic serves trust: marble, gold, and emerald styling should frame the workflow, not obscure it.

## Accessibility & Inclusion

Target WCAG AA contrast and keyboard-accessible controls. Respect reduced-motion preferences. Favor familiar controls, readable labels, and visible focus states over custom decorative interactions.
